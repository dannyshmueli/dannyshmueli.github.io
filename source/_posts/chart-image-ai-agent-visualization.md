I run an [OpenClaw](https://github.com/openclaw/openclaw) agent on a small Fly.io machine. It monitors crypto prices, prediction markets, and developer metrics ‚Äî and sends me updates on Discord and Telegram throughout the day.

The problem? It had no way to *show* me data visually.

<!-- more -->

## The Problem

My agent runs on a 256MB Fly.io instance. When I asked it to chart a token's price history, the options were bleak:

- **Puppeteer/Playwright** ‚Äî Headless Chrome needs 500MB+ RAM. My machine couldn't even install it.
- **Cloud charting APIs** ‚Äî Per-image pricing adds up fast when your agent generates dozens of charts daily.
- **Canvas libraries** ‚Äî Native dependencies that fail to compile on minimal containers.
- **Just send numbers** ‚Äî A wall of text at 3am doesn't tell you much.

I needed something that could generate a chart PNG in under a second, with zero native dependencies, on the cheapest VM money can buy.

## The Solution

**chart-image** is an [OpenClaw skill](https://clawhub.com) that generates publication-quality chart images using [Vega-Lite](https://vega.github.io/vega-lite/) and [Sharp](https://sharp.pixelplumbing.com/). No browser. No native compilation. Just Node.js.

My agent passes in data, gets back a PNG. That's it.

It supports 9 chart types, dark mode, reference lines, multi-series data, sparklines, and more ‚Äî everything an AI agent needs to turn raw data into visual insights.

**ü™ô Crypto Price Alerts**
